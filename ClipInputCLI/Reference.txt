ClipInput [file_path]
  -ratio
  -aspectRatio
    [x]:[y] (default: 16:9)
      ratio for images and triangles to correctly resize for the render settings
  -scale
    [scale] or [scaleX],[scaleY] (default: 0.5)
      scale of the whole visualizer
  -space
  -spacing
    [space] or [spaceX],[spaceY] (default: 1.05)
      breathing space for each element
  -pos
  -position
    [x],[y] (default: 0,0.5)
      position of the whole visualizer from center
  -showAfterInteraction
    [true/false] (default: false)
      determines if each input should appear only after being used
  -padOffset
    [x],[y] (default: 0.385,0)
      tells how far from each other should the base pads be
  -padColor
    [r],[g],[b],[a] (default: 0.11,0.44,0.69,1)
      color of the pad
  -padBrakeColor
    [r],[g],[b],[a] (default: 0.69,0.18,0.11,1)
      color of the analog brake
  -padBackgroundColor
    [r],[g],[b],[a] (default: 0.3,0.3,0.3,0.3)
      background color of the pad
  -padStartPos
  -padStartPosition
    [x],[y],[z] (default: 0.16,-0.45,0)
      starting symmetric point of the triangle
  -padEndPos
  -padEndPosition
    [x],[y],[z] (default: 0.6,0,0)
      ending meeting point of the triangle
  -theme
    [black/white] (default: black)
      visual style of the input visualization
  -start
  -startOffset
    [seconds] (default: 0)
      delay in second before the whole visualizer starts
  -adjustToFps
    [true/false] (default: false)
      if the activation state should be at least a frame long
  -fps
    [num] (default: 30)
      expected FPS for calculating minimal activation time

Usage: ClipInput [file_path] -scale 0.3 -pos 0.5,0.5
